# 变更记录弹窗

点击用户列表"变更记录"按钮，弹出弹窗。宽度960px，最大高度80vh。

## Tab

5个一级Tab，右侧显示待生效记录数：

| Tab | 说明 |
|---|---|
| 转移BD | BD整团队转移至新BD |
| 更换BD | 代理换归属BD |
| 更换代理 | 主播换归属代理 |
| 升为BD | 代理升级为BD |
| 升为代理 | 主播升级为代理 |

默认选中"转移BD"，切换Tab动态展示数据，取消操作后计数自动更新。

## 卡片

### 转移BD / 更换BD / 更换代理

左右双列对比：变更前（蓝色左边框） → 变更后（绿色左边框），值不同自动标红。

**转移BD**

| 变更前 | 变更后 |
|---|---|
| BD UID | BD UID |
| BD昵称 | BD昵称 |
| 所属团队ID | 所属团队ID |

**更换BD**

| 变更前 | 变更后 |
|---|---|
| 代理UID | 代理UID |
| 代理昵称 | 代理昵称 |
| 所属BD UID | 所属BD UID |
| 所属BD昵称 | 所属BD昵称 |
| 所属团队ID | 所属团队ID |

**更换代理**

| 变更前 | 变更后 |
|---|---|
| 主播UID | 主播UID |
| 主播昵称 | 主播昵称 |
| 所属代理UID | 所属代理UID |
| 代理昵称 | 代理昵称 |
| 所属团队ID | 所属团队ID |

### 升为BD / 升为代理

单块卡片，字段同列，变化标红。

**升为BD**

| 字段 | 标红 |
|---|---|
| 用户UID | - |
| 用户昵称 | - |
| 身份变更 | ✓ |
| 所属团队ID | ✓ |

身份变更格式：主播(代理) → 主播(BD)，所属团队ID标红（代理升BD独立新团队）

**升为代理**

| 字段 | 标红 |
|---|---|
| 用户UID | - |
| 用户昵称 | - |
| 身份变更 | ✓ |
| 所属团队ID | - |

身份变更格式：主播 → 主播(代理)，所属团队ID不标红（继承原BD，团队不变）

## 公共

每张卡片底部：生效时间、操作人、取消按钮。

## 操作

点击"取消"→ 内联确认弹窗 → 确认后移除卡片，Tab计数减1。

## 规则

- 仅显示待生效记录
- 滚动交互，不分页
- 转移BD/更换BD团队ID必变，更换代理JS自动判断

## 身份体系

BD > 代理 > 主播，同团队。团队归属由BD决定，一个团队1个BD。

显示格式：最低级身份(最高级身份)，如 主播(BD)、主播(代理)。
